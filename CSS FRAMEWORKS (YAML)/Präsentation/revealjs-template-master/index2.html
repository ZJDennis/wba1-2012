Was sind CSS Frameworks?

  • Werkzeug zur Layoutentwicklung
  • projektunabhängiges Baukastensystem

CSS-Frameworks sind leistungsfähige Entwicklungsumgebungen für die Erstellung von Webseiten und richten sich vorrangig an professionelle Webentwickler. Sie stellen universelle Layoutbausteine bereit und definieren ein Regelwerk, im Rahmen dessen der Entwickler sich frei bewegen kann.

Dirk Jesse, YAML Entwickler

Vorteile

  • Steigerung der Produktivität
  • Browserbugs umgehen
  • Browserübergreifende Lauffähigkeit
  • Einheitliche Code-Basis
  • verbesserter Workflow im Team

Nachteile

  • aufgeblähter Code (Overhead)
  • Einarbeitungszeit
  • Fehler anderer übernehmen

Blueprint

Blueprint

Vorteile

  • Browserübergreifend
  • veränderbare Typographie
  • Layouts sind druckfertig

Blueprint

Vorteile

  • grid
    • senkrechtes Raster
    • max 24 Spalten
    • max 950 px
    • immer 10 px rechten Rand

Blueprint

Nachteile

  • fixes Layout
  • maximale Spaltenanzahl

YAML

YAML

Aufbau, Arbeitsweise

  • Sammlung von CSS Dateien
  • Core Ordner = Basis CSS Dateien und IE Hacks
  • verschiedene zusätzliche Ordner für
  • Navigation, Formulare, Printversionen von Webseiten
  • man layoutet Webseite mittels vorhandener YAML Klassen

YAML

Features

  • Flexible, fixe und elastische Layouts
  • flexibles Grid- und Formular-System
  • sehr umfangreiche Dokumentation
  • HTML5/CSS3 ready
  • unterstützt jQuery
  • browserübergreifende Layouts

YAML

einige Klassen

  • .ym-wrapper (Layout)
  • .ym-grid (Grid)
  • .ym-column (Spalten)
  • .ym-form (Formulare)
  • .ym-clearfix (Float Handhabung)

YAML

Wrapper

  • .ym-wrapper
  • Layout-Modul
  • Standardwerte Breite
  • optinal: .ym-wbox (für padding und borders)

YAML

Grid

  • .ym-grid
  • flexibles Raster-Modul
  • Verschachtelungen möglich
  • basiert auf Prozentsätzen
  • .ym-g20, .ym-g20, .ym-g33, .ym-g50, ...
  • Anpassungen / Erweiterungen möglich

YAML

Spalten

  • .ym-column
  • teilt Inhaltsbereich in 2 bzw 3 Spalten auf
  • Breiten anpassbar (px, em, %)
  • Reihenfolge variabel
  • optional: .ym-cbox-(left / right)

YAML

Formulare

  • .ym-form
  • optional: .ym-full, .ym-required,...
  • Kind-Klassen: .ym-fbox-text, .ym-fbox-select, .ym-fbox-check,...
  • visuelle Anpassung: .ym-columnar, .ym-full, .ym-inline
  • Pflichtangaben, Fehlermeldungen, Buttons möglich

YAML

Float Handhabung

  • .ym-clearfix = float löschen
  • .ym-contain-dt = vehindert abschneiden von übergroßen Kindelementen
  • .ym-contain-oh = schneidet bei Overflow Kindelement ab und zeigt ohne Scrollbalken an
  • .ym-contain-fl = bei relativer Positionierung

YAML

Navigation

  • horizontale Navigation
  • .ym-hlist
  • vertikale Navigation
  • .ym-vlist

Aufgabe 1

http://www.thinkintags.com

Bauen Sie mit mittels "Thinkin´Tags" eine simple Web-Seite. Dazu öffnen Sie ein neues Projekt und benennen es beliebig. Als Code Base soll YAML CSS FRAMEWORK ausgewählt werden. Die Seite soll über einen Header, eine Navigation, eine Main und einen Footer verfügen.
Hinweis: Nutzen Sie den Element-Browser. Und immer mal wieder in den Source Reiter schauen, dort sieht man wie YAML im HTML Code aussieht.

Aufgabe 2

Fügen Sie die passenden Layout Einstellungen hinzu:

  • max. Seitenbreite = 600px
  • Seitenausrichtung = center
  • Header, Navigation, Main, Footer Padding = 10px

Aufgabe 3

Fügen Sie...

  • eine Headline,
  • eine horizontale Navigation,
  • ein Bild und einen Blindtext ein (mehrere Lösungen möglich),
  • zwei (neben einander) Texte mit H2 Überschrift ein (.ym-grid nutzen),
  • einen kleinen Einzeiler in den Footer ein

Aufgabe 4

Ändern Sie die Schriftart der Texte auf Verdana und die Größe auf 14pt.
Die Headlines sollen auf Impact geändert werden.

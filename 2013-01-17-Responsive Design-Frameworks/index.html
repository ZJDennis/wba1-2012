Responsive Design Framework

Grundgedanke

  • Dynamische Webseiten für verschiedene Endgeräte
  • Ansammlung von CSS-Klassen
  • Mobile Endgeräte sind die Zukunft
    • Optimierung unumgänglich

Grid-System

"herunterbrechen" auf jeweiliges Endgerät

  • Spalten
  • Drittel
  • Zwei Hälften

Frameworks

  • Skeleton
  • Twitter Bootstrap
  • Foundation 3
  • Responsive Grid System
  • ...

Vorgefertigte Elemente

  • Grundelemente in der Optik und Reaktion
  • Vorteile: Basis für Responsive Design
  • Nachteile: Komplexe Designs = extra Anpassung

Vorteile RWD

  • Anpassung der Website an jedes Gerät
  • Information steht im Vordergrund
  • Frameworks optimiert an alle gängigen Browser
  • zukunftsorientiert

Nachteile RWD

  • Manche Gestaltungselemente werden nicht unterstützt
  • Komplexe Gestaltungen schwer umsetzbar
  • Mehr Arbeit als herkömmliche Website

Media-Queries


/* Smaller than standard 960 (devices and browsers) */
@media only screen and (max-width: 959px) {}

/* Tablet Portrait size to standard 960 (devices and browsers) */
@media only screen and (min-width: 768px) and (max-width: 959px) {}

/* All Mobile Sizes (devices and browser) */
@media only screen and (max-width: 767px) {}

/* Mobile Landscape Size to Tablet Portrait (devices and browsers) */
@media only screen and (min-width: 480px) and (max-width: 767px) {}

/* Mobile Portrait Size to Mobile Landscape Size (devices and browsers) */
@media only screen and (max-width: 479px) {}

Teil 1: Was ist SASS?

Was ist SASS?

  • 2007 von Hampton Catlin, Nathan Weizenbaum und Chris Eppstein eigeführt
  • SASS ist eine Metasprache zu CSS
    • d.h. SASS-Dateien werden in standard CSS-Dateien übersetzt
  • Erweitert CSS3 mit "programmiertechnischen" Funktionen wie:
    • Variables
    • Nesting (Verschachtelung)
    • Mixins (Vererbung)

 

 

Teil 2: Warum SASS?

Warum SASS/SCSS?

Nachteile von normalem CSS

  • Eigenschaften wiederholen sich häufig
  • Umständlich bei Änderungen von Angaben, welche mehrfach an verschiedenen stellen auftauchen
  • Elemente besitzen oft dieselben Deklarationen mit gleichen Werten.
  • Selektoren können sehr lang und unübersichtlich werden.

Warum SASS/SCSS?

Vorteile und Fähigkeiten von SASS

  • Angaben können zentral definiert werden. (z.B. Farben, Abstände etc.)
  • SASS kann rechnen (z.B. Farben und Abstände relativ zu einem Wert berechnen)
  • Übersichtlicher und strukturierter
    • Elemente können von anderen Eigenschaften erben
    • Selektoren können geschachtelt werden.
  • Folglich: erhöhte Wartbarkeit

 

 

Teil 3: Unterschiede zwischen SASS und SCSS

SASS gibts es in zwei Dialekten: 1. SASS und 2. SCSS

  • SASS - Syntactically Awesome Stylesheet
    • Ältere Variante
    • Verzichtet auf Klammern und Semikolons
    • Achtet stattdessen auf die Einrückung des Codes
    • Dateiendung: .sass

SASS gibts es in zwei Dialekten: 1. SASS und 2. SCSS

  • SCSS - Sassy Cascading Stylesheet
    • Neuere und zumeist genutzte Variante
    • Obermenge zur CSS3-Systax
    • d.h. jede gültige CSS3-Syntax ist gültige SCSS-Syntax.
    • Dateiendung: .scss
  • Funktional sind SASS und SCSS gleich.
  • Wir betrachten im folgenden nur SCSS!

Kurzes Code-Beispiel

SASS-Syntax                                  SCSS-Syntax


$blue: #3bbfce                              $blue: #3bbfce;
$margin: 16px                               $margin: 16px;

.border                                     .border {
  padding: $margin / 2                        padding: $margin / 2;
  margin: $margin / 2                         margin: $margin / 2;
  border-color: $blue                         border-color: $blue;
                                            }

 

 

Teil 4: Wie kann ich SASS/SCSS benutzen?

Variablen


// Farben
$textcolor:#4f4f4f;
$navcolor:#DFE1CB;
// Fonts
$textfont:arial, helvetica, san-serif;
$fontsize:12px;
// Sizes
$marginleft:90px;
$marginright:$marginleft + 5px;
  • Werden mit $VARNAME: VALUE; definiert.
  • Mit numerischen Variablen kann gerechnet werden. (+, -, *, /)

Nesting / Verschachtelung


nav {
    width:330px;
    margin-left:0;
    margin-right:0;
    ul {
        list-style-type:none;
        margin:0;
        padding:0;
    }
    li {
        float:left;
        background-color:$navcolor;
        margin:0px;
        padding:0px;
    }
}
  • Tags können wie im HTML verschachtelt werden.

Nesting / Verschachtelung

Aus vorangegangener .scss ensteht folgende .css 


nav {

    margin-left:0;
    margin-right:0; 
}

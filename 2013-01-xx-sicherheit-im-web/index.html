Browser und Sicherheit

Cookies

  • Enthalten Identifikationscode zum Identifizieren eines Busuchers
  • Realisiert Sitzungen, Einkaufskörbe,...
  • Erstellen von Benutzerprofilen (Spionage)
  • Cookie-Stealing = "Identitätsdiebstahl"
  • Lösung: Cookies ganz/teilweise deaktivieren, https Verschlüsselung verwenden

Browser und Sicherheit

Zertifikate

  • Verschlüsselte Übertragung
  • Garantiert Identität der Webseite
  • Zertifikataustellung durch Certificate Authenticity

Schwierigkeiten

  • Gefälschte Zertifikate
  • 'Man-in-the-Middle Zertifikate'
  • Selbst erstellte Zertifikate

Verschlüsselung

  • Klartext -> Geheimtext
  • Symmetrische Verschlüsselung
    • Ein geheimer Schlüssel
    • Schnell aber unsicher
  • Asymmetrische Verschlüsselung
    • Öffentlicher + Geheimer Schlüssel
    • Langsamer aber sicherer
    • Erfordert Rechenaufwand

Hash-Funktionen

  • Kryptographische Hash-Funktionen
    • SHA256
    • SHA512
    • WHIRLPOOL
    • ...
  • Vorteile
    • "One-Way-Functions"
    • hallo = 598d4c200461b81522a3328565c25f7c
hello = 5d41402abc4b2a76b9719d911017c592
    • Kollisionsresistent

Hash-Funktionen

  • Hintergrund:
    • Passwort für Angreifer unbrauchbar machen
    • Passwort wird nie gespeichert!
    • Hashcode wird gespeichert, der keine Rückschlüsse auf Passwort erlauben (soll)
  • Nachteile:
    • Sicherheit heutzutage nicht mehr gegeben
    • Riesige Datenbanken im Netz, mit vorberechneten Hashes
    • gleiche Hashwerte bei gleichem Passwort
    • Lösung: Salting

Salting

    • Salt = zufällige Zeichenkette
    • Wird mit Passwort konkateniert
    • hallo + 565c25f7c = 3d8a60305864ca1300d4b67c58c82c94
hallo + b9719d91f = 9fc0c1698b0cda1f3a2037e9723312ca
    • Wichtig:
      • Ein Salt nur einmal benutzen!
      • Salt sollte groß und zufällig sein!

Salting: Speichern und Valideren

  1. Großen, zufälligen Salt generieren
  2. Salt mit Passwort konkatenieren und hashen
  3. Hash und Salt in der Datenbank abspeichern
  4. Bei Validierung wird Salt aus Datenbank mit dem übermittelten Passwort gehasht
  5. Ergebnis-Hash mit Hash aus Datenbank vergleichen
    • Übereinstimmung = Zugang

XSS Injection

Formularfeld Ausgabe falsch

Formularfeld Ausgabe richtig


'Ihre Suche nach "'.htmlspecialchars($suche).'" ergab folgende Treffer:'
  • Textfeld Size begrenzen!

SQL Injection

Code falsch


$email = $_POST['username'];
$password = $_POST['password'];
$query = "SELECT `kundennummer` FROM `user` WHERE `passwort` = '$password' AND `email` = '$email'";

Code richtig


$email = trim(mysql_real_escape_string($_POST['username']));
$password = trim(mysql_real_escape_string($_POST['password']));
$query = "SELECT `kundennummer` FROM `user` WHERE `passwort` = '$password' AND `email` = '$email'";

Wie funktioniert eine Suchmaschine?

  • Versucht das surfverhalten der Nutzer nachzubilden
  • Crawler rufen Seiten automatiert & regelmäßig ab
  • Der Indexierungsvorgang interpretiert das Markup ...
    (Implementierung i.d.R. Geschäftsgeheimnis)
  • ... und die semantische Bedeutung dessen:
    • Bewertet die Wichtigkeit (<title>, <h1>, <small>, ...)
    • Folgt links für eine weitere Indizierung (<a> und <area>)

Content: Grundlagen

  • Bewusstsein verschiedener Zielgruppen
    • Neulinge
    • Kenner
  • Saubere Sprache
  • Strukurierter Text

Content: Ankertext

  • Kurz, aussagekräftig & zusammenhängend
  • Selbst verfasst
  • Sinnvolle interne Links für den Nutzer - nicht für Suchmaschienen!

Content: Don'ts

  • Text in Bildern
  • Wiederholender Inhalt
  • Text für Suchmaschienen schreiben

Struktur: Domainname & URL

  • Worte benutzen
  • Einfache Verzeichnis-Struktur Beispiel
  • Eine URL pro Dokument Negativ Beispiel

Struktur: Titel-Tag

  • Innerhalb des head-Tags
  • Üblicherweiße ähnlich der <h1>
  • Erste Zeile der Suchergebnisse
...
<head>
    <title>Achsen - Skateshop24</title>
</head>
<body>
    <h1>Achsen</h1>
    ...

Struktur: Titel-Dos

  • Content der Seite beschreiben
  • Einen Titel für jede Seite
  • Kurz aber Aussagekräftig

Struktur: Meta-Tag

  • "description" beschreibt die Seite
  • benutzt, falls kein sinnvoller Absastz vorhanden ist
... 
<head>
    <title>Achsen - Skateshop24</title>
    <meta name="description" content="Auf Skateshop24 findest du rund um die Uhr eine riesige Auswahl an Achsen für dein Skateboard" />
</head>
...

Struktur: Meta-Tag Don'ts

  • Die ganze Seite reinkopieren
  • Mit Keywords vollstopfen
  • Ständige Verwendung der selbem Description

Struktur: Bilder

  • Inhaltlich passende Titel - trotzdem kurz
  • "alt" Attribut nutzen
  • Ein Verzeichnis für alle Bilder
  • Gebräuchliche Formate
<img src="../images/Rollen57.jpg" alt="Rollen 57 - Bigfoot"/>

Struktur: Bilder Don'ts

  • Standard Dateinamen
  • Lange Alt-Texte - kann als Spam angesehen werden
  • Voll mit Keywords

Crawler steuern: Anker-Tags

  • Links können Steuerbefehle für Suchmaschinen beinhalten
  • „nofollow“ verhindert das die Zieladresse indiziert wird
<nav>
	<a href="/">Home</a>
	<a href="/mitarbeiter.html" rel="nofollow">Mitarbeiter</a>
	<a href="/kontakt.html">Kontakt</a>
	...

Crawler steuern: Meta-Tags

  • Meta-Tag definiert Steuerbefehle Dokumentenweit
  • „noindex“ verhindert indexierung der aktuellen Seite
  • „nofollow“ legt Standard für Anker-Tags fest
  • Kann pro Anker-Tag überschrieben werden
<head>
	<meta name="robots" content="noindex, nofollow" />
	...
</head>
<body>
	...
	<a href="/" rel="follow">Home</a>
	<a href="/mitarbeiter.html">Mitarbeiter</a>
	<a href="/kontakt.html" rel="follow">Kontakt</a>

Crawler steuern: Robots Exclusion Protocol

  • Text-Datei mit Regeln zur Suchmaschinen-Steuerung
  • Per Konvention wird der Pfad /robots.txt verwendet
User-agent: *
Disallow: /mitarbeiter.html
Disallow: /images/
